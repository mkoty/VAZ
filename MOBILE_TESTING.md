# Тестирование с мобильного устройства

## Проблема
При тестировании на реальном мобильном устройстве (iPhone/Android), приложение не может подключиться к `localhost:3001`, потому что `localhost` на телефоне указывает на сам телефон, а не на ваш компьютер.

## Решение

### 1. Узнайте IP адрес вашего компьютера

**macOS:**
```bash
ipconfig getifaddr en0
# Или
ifconfig | grep "inet " | grep -v 127.0.0.1
```

**Windows:**
```bash
ipconfig
# Найдите IPv4 Address в разделе Wi-Fi
```

Пример: `192.168.1.100`

### 2. Настройте Backend для прослушивания всех интерфейсов

В `backend/src/main.ts` измените:
```typescript
await app.listen(3001, '0.0.0.0'); // Вместо просто 3001
```

### 3. Обновите CORS настройки

Backend уже настроен на разрешение запросов с любых origin в режиме разработки.

### 4. Запустите Backend

```bash
cd backend
npm run dev
```

Backend должен показать:
```
Nest application successfully started
Listening on http://0.0.0.0:3001
```

### 5. Запустите Frontend с вашим IP

```bash
cd ..
NEXT_PUBLIC_API_URL=http://192.168.1.100:3001 npm run dev
```

Замените `192.168.1.100` на ваш реальный IP адрес.

### 6. Откройте на телефоне

**Важно**: Телефон и компьютер должны быть в одной Wi-Fi сети!

Откройте в браузере телефона:
```
http://192.168.1.100:3000
```

Замените `192.168.1.100` на ваш IP адрес.

## Альтернатива: Используйте ngrok

Если не хотите возиться с IP адресами:

### 1. Установите ngrok
```bash
brew install ngrok  # macOS
# Или скачайте с https://ngrok.com
```

### 2. Запустите backend туннель
```bash
ngrok http 3001
```

Вы получите публичный URL, например: `https://abc123.ngrok.io`

### 3. Запустите frontend туннель (в другом терминале)
```bash
ngrok http 3000
```

Вы получите публичный URL для frontend, например: `https://def456.ngrok.io`

### 4. Обновите переменные окружения

Создайте `.env.local`:
```
NEXT_PUBLIC_API_URL=https://abc123.ngrok.io
```

### 5. Перезапустите frontend
```bash
npm run dev
```

### 6. Откройте на телефоне
```
https://def456.ngrok.io
```

## Проверка

Если все работает правильно, вы должны увидеть:
- Приложение загружается на телефоне
- После нажатия "Получить код" появляется индикатор загрузки
- Код приходит на email
- Нет ошибок в консоли браузера

## Отладка

Если код не приходит:
1. Откройте консоль разработчика на телефоне (Safari: Настройки → Safari → Advanced → Web Inspector)
2. Проверьте логи backend на компьютере
3. Убедитесь что firewall не блокирует порты 3000 и 3001
