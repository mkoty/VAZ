## Предварительные требования

✅ У вас уже развернуты:
- Backend NestJS приложение
- PostgreSQL база данных

❗ Теперь нужно развернуть:
- Frontend Next.js приложение

## Шаг 1: Создание нового приложения для Frontend

1. Зайдите в панель управления https://timeweb.cloud/
2. Перейдите в раздел **"Cloud Apps"** или **"Приложения"**
3. Нажмите **"Создать приложение"**
4. Выберите:
   - **Название**: `avtovaz-frontend` (или любое другое)
   - **Язык**: Node.js
   - **Версия**: 18 или 20
   - **Репозиторий**: Подключите GitHub репозиторий
   - **Ветка**: `main`
   - **Рабочая директория**: `/` (корень проекта)

## Шаг 2: Настройка переменных окружения Frontend

В настройках приложения добавьте переменные окружения:

```env
NODE_ENV=production
DEPLOY_TARGET=timeweb
NEXT_PUBLIC_API_URL=https://mkoty-vaz-8c2f.twc1.net
```

**Важно**: Замените `https://mkoty-vaz-8c2f.twc1.net` на реальный URL вашего backend приложения!

### Как получить URL backend:
1. Откройте ваше backend приложение в панели Timeweb
2. Скопируйте URL вида `mkoty-vaz-8c2f.twc1.net`
3. Вставьте его в `NEXT_PUBLIC_API_URL`

## Шаг 3: Настройка команд сборки и запуска

В настройках приложения укажите:

### Build Command (команда сборки):
```bash
npm run build:timeweb
```

### Start Command (команда запуска):
```bash
npm start
```

### Port (порт):
```
3000
```

### Working Directory (рабочая директория):
```
/
```
(корень проекта, НЕ `/backend`)

## Шаг 4: Деплой

1. Нажмите кнопку **"Развернуть"** или **"Deploy"**
2. Дождитесь завершения сборки (3-5 минут)
3. После успешного деплоя получите URL вида:
   ```
   https://avtovaz-frontend.timeweb.cloud
   ```

## Шаг 5: Обновление CORS на Backend

Важно! После получения URL frontend, обновите настройки CORS в backend:

1. Откройте настройки вашего backend приложения на Timeweb
2. Добавьте переменную окружения:
   ```env
   FRONTEND_URL=https://ваш-frontend.timeweb.cloud
   ```

3. Убедитесь, что в `backend/src/main.ts` есть корректная настройка CORS:
   ```typescript
   app.enableCors({
     origin: [
       'http://localhost:3000',
       process.env.FRONTEND_URL || 'https://ваш-frontend.timeweb.cloud'
     ],
     credentials: true,
   });
   ```

## Шаг 6: Проверка работоспособности

1. Откройте URL вашего frontend приложения
2. Проверьте, что:
   - ✅ Страницы загружаются
   - ✅ API запросы идут на правильный backend URL
   - ✅ Нет ошибок CORS в консоли браузера

## Troubleshooting

### ❌ Ошибка: "API requests failing"

**Причина**: Неправильный `NEXT_PUBLIC_API_URL`

**Решение**:
1. Проверьте переменную окружения `NEXT_PUBLIC_API_URL`
2. Убедитесь, что backend доступен по этому URL
3. Проверьте CORS настройки на backend

---

### ❌ Ошибка: "CORS policy blocked"

**Причина**: Backend не разрешает запросы с frontend домена

**Решение**:
1. Добавьте URL frontend в `FRONTEND_URL` на backend
2. Проверьте настройки CORS в `backend/src/main.ts`
3. Перезапустите backend приложение

---

### ❌ Ошибка при сборке: "Module not found"

**Причина**: Не установлены зависимости

**Решение**:
1. Проверьте, что `package.json` в корне проекта
2. Убедитесь, что build command правильный: `npm run build:timeweb`
3. Проверьте логи сборки на наличие ошибок установки пакетов

---

### ❌ Приложение не запускается

**Причина**: Неправильный порт или start command

**Решение**:
1. Убедитесь, что Port = `3000`
2. Проверьте Start Command: `npm start`
3. Убедитесь, что `next.config.ts` настроен правильно

---

### ❌ Статические файлы не загружаются

**Причина**: Неправильная конфигурация `next.config.ts`

**Решение**:
1. Убедитесь, что `DEPLOY_TARGET=timeweb` установлен
2. Проверьте, что `output: 'standalone'` используется
3. Проверьте, что `basePath` не установлен для timeweb
