# 📧 Настройка Email на Railway

## Проблема
Локально коды приходят, а на Railway (production) - нет.

## Причина
На Railway не настроены переменные окружения для SMTP:
- `EMAIL_USER`
- `EMAIL_PASSWORD`

## Решение

### Шаг 1: Откройте Railway Dashboard

1. Перейдите на https://railway.app
2. Войдите в аккаунт
3. Откройте проект с backend

### Шаг 2: Добавьте переменные окружения

1. В проекте откройте ваш backend сервис
2. Перейдите на вкладку **Variables**
3. Добавьте следующие переменные:

#### Для Yandex Mail:

```
EMAIL_USER=ваша-почта@yandex.ru
EMAIL_PASSWORD=ваш-пароль-приложения
```

**Как получить пароль приложения Yandex:**
1. Откройте https://id.yandex.ru/security/app-passwords
2. Введите название приложения (например, "LADA Backend")
3. Скопируйте сгенерированный пароль (НЕ обычный пароль от почты!)

#### Для Gmail:

```
EMAIL_USER=ваша-почта@gmail.com
EMAIL_PASSWORD=ваш-app-password
```

**Как получить App Password в Gmail:**
1. Откройте https://myaccount.google.com/apppasswords
2. Включите двухфакторную аутентификацию (если не включена)
3. Создайте App Password для "Mail"
4. Скопируйте 16-значный пароль

### Шаг 3: Сохраните и подождите

После добавления переменных:
1. Railway автоматически перезапустит сервис
2. Подождите 1-2 минуты

### Шаг 4: Проверьте логи

1. В Railway откройте вкладку **Deployments**
2. Кликните на последний деплой
3. Откройте **View Logs**
4. Найдите сообщение:

✅ **Успешно:**
```
✅ SMTP сервер готов к отправке писем
```

❌ **Ошибка:**
```
❌ Ошибка подключения к SMTP: ...
⚠️ EMAIL_USER или EMAIL_PASSWORD не настроены
```

### Шаг 5: Протестируйте

1. Откройте ваше приложение на GitHub Pages или локально
2. Попробуйте запросить код
3. Проверьте почту - код должен прийти!

## Отладка

### Проверка текущих переменных

В Railway Dashboard → Variables проверьте:
- ✅ `EMAIL_USER` установлен
- ✅ `EMAIL_PASSWORD` установлен
- ✅ Нет опечаток в email
- ✅ Используется пароль приложения (НЕ обычный пароль)

### Проверка логов отправки

В логах Railway ищите:
```
🔍 Поиск пользователя email=test@example.com: не найден ❌
♻️  Повторная отправка существующего кода для test@example.com
✅ Код отправлен на email test@example.com
```

### Частые ошибки

1. **Invalid login: 535 5.7.8 Authentication failed**
   - Используется обычный пароль вместо пароля приложения
   - Решение: Создайте пароль приложения (см. выше)

2. **Connection timeout**
   - Railway не может подключиться к SMTP серверу
   - Решение: Проверьте, что не заблокированы порты (обычно не проблема на Railway)

3. **Sender address rejected**
   - Неправильный формат EMAIL_USER
   - Решение: Проверьте, что EMAIL_USER - это полный email адрес

## Альтернативные решения

Если Yandex/Gmail не работают, можно использовать:

### 1. SendGrid (бесплатно до 100 писем/день)

```bash
# Установите пакет
npm install @sendgrid/mail

# Добавьте в Railway Variables:
SENDGRID_API_KEY=ваш-api-ключ
```

### 2. Mailgun (бесплатно до 5000 писем/месяц)

```bash
# Установите пакет
npm install mailgun.js

# Добавьте в Railway Variables:
MAILGUN_API_KEY=ваш-api-ключ
MAILGUN_DOMAIN=ваш-домен
```

## Проверка что всё работает

### 1. В Railway логах должно быть:
```
🚀 API Server running on http://0.0.0.0:3001
✅ SMTP сервер готов к отправке писем
```

### 2. При запросе кода:
```
🔍 Поиск пользователя email=test@example.com: не найден ❌
🆕 Сгенерирован новый код для test@example.com
✅ Код отправлен на email test@example.com
Message ID: <...@yandex.ru>
```

### 3. На почте:
Письмо с темой "Код подтверждения LADA ID" и 4-значным кодом.

## Что дальше?

2. ✅ На Railway работает → с теми же учетными данными
3. ✅ На GitHub Pages работает → подключается к Railway backend
