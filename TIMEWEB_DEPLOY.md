# Деплой на Timeweb Cloud

## Шаг 1: Регистрация
1. Перейдите на https://timeweb.cloud/
2. Зарегистрируйтесь (можно с российским номером)
3. Пополните баланс на ~500-1000₽

## Шаг 2: Создание PostgreSQL базы данных

1. В панели управления Timeweb Cloud выберите **"Базы данных"**
2. Нажмите **"Создать базу данных"**
3. Выберите:
   - Тип: **PostgreSQL**
   - Версия: **14 или выше**
   - Тариф: **Базовый** (~200₽/месяц)
4. Сохраните данные подключения:
   - Host
   - Port
   - Database name
   - Username
   - Password

## Шаг 3: Создание Node.js приложения

1. В панели выберите **"Cloud Apps"** или **"Приложения"**
2. Нажмите **"Создать приложение"**
3. Выберите:
   - Язык: **Node.js**
   - Версия: **18 или 20**
   - Репозиторий: Подключите GitHub репозиторий `mkoty/VAZ`
   - Ветка: **main**

## Шаг 4: Настройка переменных окружения

В настройках приложения добавьте переменные:

```env
NODE_ENV=production
DEPLOY_TARGET=timeweb
DATABASE_URL=postgresql://username:password@host:port/database_name
```

## Шаг 5: Настройка сборки

В настройках приложения:
- **Build Command**: `npm run build:timeweb`
- **Start Command**: `npm start`
- **Port**: `3000`

## Шаг 6: Деплой

1. Нажмите **"Развернуть"** или **"Deploy"**
2. Timeweb автоматически:
   - Склонирует репозиторий
   - Установит зависимости
   - Соберет приложение
   - Запустит сервер

## Шаг 7: Проверка

После успешного деплоя вам будет предоставлен URL вида:
`https://avtovaz-reception.timeweb.cloud` или аналогичный

## Автоматический деплой

После настройки, каждый push в ветку `main` будет автоматически деплоиться на Timeweb Cloud.

## Полезные команды

```bash
# Просмотр логов (если доступен CLI)
timeweb logs avtovaz-reception

# Перезапуск приложения
timeweb restart avtovaz-reception
```

## Troubleshooting

### Ошибка при сборке
- Проверьте логи сборки в панели Timeweb
- Убедитесь что `DEPLOY_TARGET=timeweb` установлен

### Приложение не запускается
- Проверьте что порт 3000 правильно настроен
- Убедитесь что `npm start` работает локально

### База данных не подключается
- Проверьте правильность `DATABASE_URL`
- Убедитесь что БД создана и доступна
